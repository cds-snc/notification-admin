The badge component is used to highlight new or important information. It can be used with or without an icon.

Moderate use of this component is key to its success, if there are too many badges, they become less effective in attracting attention.

The badge has two parts:

  1. The label (required)
  2. The icon (optional)
  3. The icon meaning (optional)

When using an icon, you need to think about what it represents. It should usually act as a way to shorten a prefix to the label, such as "Category: General communication". Sometimes, you need no prefix at all.

The badge component will not render an icon if you don't have an icon label, as this is important for screen readers.

The badge component has 6 variants to choose from, mostly based on the color you need: gray, green, blue, yellow, lime, red, purple, and rose.

There are two keyword variants for success (green) and default (gray).
